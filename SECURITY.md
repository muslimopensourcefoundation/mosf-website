# Security Policy

## Reporting a Vulnerability

The Muslim Open Source Foundation takes security vulnerabilities seriously. We appreciate your efforts to responsibly disclose your findings, and we will make every effort to acknowledge your contributions.

### How to Report

If you discover a security vulnerability, please report it to us using one of the following methods:

**Email (Preferred):**
- **Security Team:** beder.bourahmah@muslimopensource.org
- **Subject Line:** `[SECURITY] Brief description of the vulnerability`

Please include the following information in your report:

1. **Type of issue** (e.g., buffer overflow, SQL injection, cross-site scripting, etc.)
2. **Full paths of source file(s) related to the manifestation of the issue**
3. **The location of the affected source code** (tag/branch/commit or direct URL)
4. **Step-by-step instructions to reproduce the issue**
5. **Proof-of-concept or exploit code** (if possible)
6. **Impact of the issue**, including how an attacker might exploit the issue

### What to Expect

- You will receive an acknowledgment of your report within **48 hours**
- We will provide a more detailed response within **7 days**, indicating the next steps in handling your report
- We will keep you informed of the progress toward resolving the issue
- After the vulnerability has been resolved, we will notify you and may credit you for the discovery (if you wish)

### Disclosure Policy

- **Do not** disclose the vulnerability publicly until we have had a chance to address it
- **Do not** access user data that is not your own
- **Do not** disrupt our services or destroy data during security testing
- **Do** provide us with a reasonable amount of time to address the vulnerability before public disclosure

### Scope

The following vulnerabilities are **out of scope** for this security policy:

- Social engineering attacks
- Physical security issues
- Denial of service attacks
- Issues that require physical access to a user's device
- Spam or unsolicited messages
- Issues that require outdated browsers or plugins

### Recognition

Security researchers who responsibly disclose vulnerabilities in accordance with this policy may be recognized in our security advisories, unless they prefer to remain anonymous.

## Security Updates

Security updates will be released as quickly as possible. We will:

1. Assess the severity of the vulnerability
2. Develop and test a fix
3. Release an update with appropriate security patches
4. Publish a security advisory detailing the vulnerability and fix (after users have had time to update)

## Questions

If you have questions about this security policy, please contact:
- **Email:** beder.bourahmah@muslimopensource.org
- **General Inquiries:** contact@muslimopensource.org
